Easy Trigger

Timecode-driven Resolume Clip Trigger – User Guide
Easy Trigger receives a timecode signal (LTC, MTC, Art-Net, or OSC), compares the current position against a list of configured clip trigger times, and fires OSC commands to Resolume Arena / Avenue to play the matching clips at the right moment. Each clip has an editable trigger time, optional countdown, duration, and an end-of-clip action. Custom Triggers allow firing arbitrary Resolume columns or layer/clip combinations without a clip list.

Timecode Input Settings

The collapsible left panel section. Select the source type from the dropdown, then configure the fields that appear for that source. Switching source type immediately restarts the listener.

LTC Input

Decodes LTC (Linear Timecode) from an audio input device.

FieldDescription
DriverAudio host API: WASAPI, DirectSound (Windows), CoreAudio (macOS), or ASIO (Windows low-latency, requires ASIO SDK build). Default (all devices) lists everything.
Device (Input)The audio input device carrying the LTC signal. Filtered by the selected driver.
ChannelInput channel to decode: 1–8 or 1+2 (stereo merge). Most rigs use a single mono channel.
Sample RateDefault uses the device's current rate. Set explicitly (44100, 48000, 96000, etc.) if the device does not auto-negotiate. Must match the device setting.
Level meterReal-time bar showing input signal strength. Should show a healthy mid-range when LTC is present. Empty bar means no audio is arriving.
Input GainGain from −24 dB to +24 dB (0.1 dB steps). Double-click to reset to 0 dB. Adjust to normalise the LTC signal level before decoding.

MTC Input

Receives MIDI Timecode from a MIDI port.

FieldDescription
MIDI InputDropdown of available MIDI input ports. Select the port sending MTC. Virtual ports from DAWs appear here too.

ArtNet Input

Receives Art-Net Timecode packets over UDP on port 6454.

FieldDescription
AdapterLocal network interface to listen on. ALL INTERFACES binds to 0.0.0.0.
Listen IPBind address derived from the adapter. Can be overridden manually.

OSC Input

Receives timecode position via OSC messages over UDP. Compatible with Reaper and other OSC-capable software.

FieldDescription
AdapterNetwork interface to bind the OSC listener.
Listen IPIP address to bind. Use 127.0.0.1 for same-machine senders, 0.0.0.0 for network-wide.
PortUDP port number (default 9000). Must match the sender's configured output port.
FPSFrame rate used to interpret incoming float time values (24 / 25 / 29.97 / 30). Match your project FPS.
String addressOSC path for string timecode messages (e.g. /frames/str). Expects HH:MM:SS:FF or HH:MM:SS.FF.
Float addressOSC path for float position messages (e.g. /time). Value is interpreted as seconds × FPS.

Resolume Settings

Configures the connection between Easy Trigger and Resolume Arena / Avenue.

FieldDescription
Output IPIP address of the machine running Resolume. Use 127.0.0.1 if Resolume is on the same computer. Otherwise enter the network IP of the Resolume machine.
Output PortUDP port that Resolume listens on for incoming OSC (default 7000). Must match the OSC Server port in Resolume's preferences.
Input PortUDP port on which Easy Trigger listens for OSC feedback from Resolume (default 7001). Must match the OSC Client port in Resolume. Used to receive the current connected-clip state so rows highlight correctly.
Sync indicatorColoured dot next to the Resolume header. Green = OSC listener is running and Resolume feedback is being received. Red / grey = listener not running or no feedback arriving.
Global OffsetA signed timecode offset applied to every trigger time before comparison. Format: ±HH:MM:SS:FF (e.g. -00:00:00:05 fires clips 5 frames early). Useful to compensate for Resolume or network latency.
In Resolume, go to Preferences → OSC and enable the OSC server on the port set in Output Port, and the OSC client pointing to Easy Trigger's machine on the Input Port.

Clip Table – Layer Headers

The right panel displays a scrollable table of all Resolume layers and clips. Each layer has a header row followed by its clip rows.

ElementDescription
▶ / ▼ arrow buttonCollapses or expands the layer's clip rows. Click to hide/show all clips in that layer without changing any settings.
Layer checkboxEnables or disables the entire layer for timecode triggering. When unchecked, no clips in that layer will fire regardless of their individual checkbox state.
Layer nameName of the Resolume layer, fetched from Resolume via the Get Clips button. Read-only — edit layer names in Resolume itself.

Clip Table – Clip Rows

Each clip row in the table represents one Resolume clip slot. Rows are coloured to show their state:

  • Green row — this clip is currently active/connected in Resolume (feedback received via OSC Input Port).
  • Orange row — this clip was fired by a timecode trigger in this session.
  • Default (dark) — inactive or not triggered.
ColumnDescription
CheckboxEnables or disables this specific clip for timecode triggering. When unchecked, the clip is skipped even if its Trigger Time matches the current timecode.
Clip nameThe clip slot name as reported by Resolume. Displayed read-only. Updated when Get Clips is used.
CountdownShows the time remaining until this clip's Trigger Time. Counts down from a positive value to zero as the timecode approaches. Displayed as −HH:MM:SS:FF. Grey when far away, becomes more visible as it approaches.
Trigger TimeThe timecode position at which this clip fires. Editable — click to edit inline. Format: HH:MM:SS:FF. The Global Offset is subtracted from this value before comparing. Leave empty or set to 00:00:00:00 to disable triggering for this clip.
DurationHow long (in timecode) this clip is expected to play. Used by the End Action to determine when to fire the post-clip action. Format: HH:MM:SS:FF. Editable inline.
End ActionAction taken when the timecode reaches Trigger Time + Duration. Cycles through three states by clicking:
Off — no action after the clip finishes.
Col — fires the Resolume column number entered in the detail field (column-based triggering).
L/C — fires a specific layer/clip combination entered as Layer and Clip numbers.
▶ Test buttonManually sends the OSC trigger for this clip immediately, regardless of current timecode. The row highlights green to confirm the command was sent. Useful for testing the Resolume connection and verifying clip assignments.

Custom Triggers

A special group (Layer 0) that appears above the regular clip layers. Custom Triggers are not tied to any specific Resolume clip slot — each one fires a column or a layer/clip combo at a configured time. Useful for triggering global Resolume events that are not individual clips.

Create the first Custom Trigger using the Create Custom Trigger button (appears in the toolbar when no custom group exists). Additional triggers are added with the + button inside the group.

Settings Menu

Opened by clicking Settings. All configuration is stored in .etr files (JSON format). Three save/load scopes are available:

ItemDescription
Save settings…Saves only the Timecode Input and Resolume connection settings (no clip data).
Save clips…Saves only the clip table (trigger times, durations, end actions) without the input/connection settings.
Save all…Saves everything: input settings, Resolume settings, clip table, and custom triggers.
Load settings…Loads and applies only the settings portion of an .etr file. Clip data in the file is ignored.
Load clips…Loads and applies only the clip table from an .etr file. Input/connection settings are not changed.
Load all…Loads and applies everything from the file.
Load on startupToggle (checkbox). When enabled, the last saved/loaded .etr file is automatically loaded at startup. The setting is persisted to an internal preferences file.
Close to trayToggle (checkbox). When enabled, closing the window minimises to the system tray instead of quitting.
Clear custom triggersRemoves all custom trigger rows and the custom trigger group. Enabled only when a custom trigger group exists.
Clear clip triggersRemoves all regular clip rows from the table (does not affect custom triggers or input settings). Use this before doing a fresh Get Clips if the Resolume composition has changed significantly.

File Format (.etr)

Easy Trigger uses JSON files with the .etr extension. The file contains a mode field ("settings", "clips", or "all") that tells the loader which sections to restore. Files created with Save settings can be loaded with Load all — the missing clip section is simply skipped.

Files are saved to the user's Documents folder by default but can be placed anywhere.

OSC Addresses Used

Easy Trigger sends the following OSC messages to Resolume when triggering clips or columns:

ActionOSC Address
Trigger clip/composition/layers/[L]/clips/[C]/connect
Trigger column/composition/columns/[col]/connect
Set clip offset/composition/layers/[L]/clips/[C]/transport/position/behaviour/offset
Set clip duration/composition/layers/[L]/clips/[C]/transport/position/behaviour/duration

Where [L] is the 1-based layer index, [C] is the 1-based clip index, and [col] is the 1-based column index.

Troubleshooting

ProblemWhat to check
Status stays STOPPEDCheck the source type and all source fields (device, port, adapter). For LTC, verify the Level meter is not empty. For MTC, confirm the correct MIDI port is selected.
Get Clips returns nothingVerify Output IP and Output Port match Resolume's OSC server settings. Verify Input Port matches Resolume's OSC client settings. Confirm Resolume is running with a composition loaded.
Clips don't trigger on timeCheck that layer and clip checkboxes are enabled. Verify Trigger Time format is HH:MM:SS:FF. Use Global Offset to compensate for Resolume latency. Test manually with the ▶ button to verify the connection works.
Row stays red / not green after triggerThe OSC feedback from Resolume is not arriving. Check Input Port is correct in both Easy Trigger and Resolume's OSC client config. Check firewall rules on the Input Port.
"OSC listener is not running"The Input Port is in use by another application, or the listener failed to start after loading a settings file. Change the port or restart the application.
Custom trigger fires wrong clipConfirm the Col or L/C numbers entered in the Type/Source field match the actual column or layer/clip index in Resolume (1-based).
LTC decode unreliableOn Windows, switch the Driver to ASIO or WASAPI Exclusive for lower jitter. Adjust Input Gain so the Level meter shows a healthy signal.
